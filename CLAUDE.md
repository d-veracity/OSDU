# CLAUDE Workflow Rules

This document establishes the operational workflow for Claude when working on the OSDU Developer project. These rules ensure a structured, collaborative, and simple approach to all tasks.

## Project Context

The OSDU Developer project provides simplified personal deployments of the OSDU™ data platform on Microsoft Azure. All work should maintain the project's focus on simplicity and reliability.

## Workflow Rules

### 1. Think and Plan First
First think through the problem, read the codebase for relevant files, and write a plan to `tasks/todo.md`.

### 2. Create Actionable Todo Lists
The plan should have a list of todo items that you can check off as you complete them. Use markdown checkboxes for clear tracking.

### 3. Verify Before Starting
Before you begin working, check in with me and I will verify the plan. No implementation should start without approval.

### 4. Execute and Track Progress
Then, begin working on the todo items, marking them as complete as you go. Update the `tasks/todo.md` file to reflect progress.

### 5. Communicate Changes Clearly
Please every step of the way just give me a high level explanation of what changes you made. Focus on the impact and purpose, not implementation details.

### 6. Prioritize Simplicity
Make every task and code change you do as simple as possible. We want to avoid making any massive or complex changes. Every change should impact as little code as possible. Everything is about simplicity.

### 7. Document and Review
Finally, add a review section to the `tasks/todo.md` file with a summary of the changes you made and any other relevant information.

## File Structure

- `tasks/todo.md` - Current task planning and progress tracking
- `CLAUDE.md` - This workflow document (you are here)

## Notes

- Always prioritize minimal impact changes
- Maintain compatibility with existing OSDU Developer functionality
- Follow the project's existing patterns and conventions
- Ask for clarification when requirements are unclear
